# Backlog Duplicate Demo Prompts

Send these prompts one message at a time. Wait for the observer to answer each
step before sending the next one.

## 1. Start The Demo

```text
Use this current Claude Code or Codex session as the observer for the Backlog
Duplicate demo. If there is no safe project ready, set up the isolated fixture
first. Show me the project id and backlog dashboard link.
```

## 2. Propose A Similar Requirement

```text
I want to add configurable default reminder settings to the planner demo.
Please check whether this overlaps existing backlog work before you create
anything new.
```

## 3. Ask For The Options

```text
Show me the similar backlog item or items you found, explain the overlap in
plain language, and ask me whether I want to merge, supersede, or keep the new
request separate.
```

## 4A. User Choice: Merge

```text
Merge this request into the existing backlog item. Keep the original context,
add the configurable reminder default detail, and show me the updated backlog
evidence.
```

## 4B. User Choice: Supersede

```text
Supersede the older backlog item with this request. Mark the old direction as
replaced, preserve the reason, and show me the updated backlog evidence.
```

## 4C. User Choice: Separate

```text
Keep this as a separate backlog item. Explain why it is separate from the
similar work and show me both items in the dashboard.
```

## 5. Ask For The Evidence Summary

```text
Summarize the Backlog Duplicate demo evidence: new request, similar backlog
items, user choice, final backlog state, dashboard links, and any limitations.
```
